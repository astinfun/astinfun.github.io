java基础

    Arrays.copyOf（array,length） 扩容

  • 7.并发：
    volatile： 内存可见性（基本类型变量，其值是直接拷贝到存在中，而volatile是直接操作变量）,不能保证并发正确性
    synchronized : 尽量锁住数据，减小锁粒度
    Lock : 锁，可重入锁等

  • 8.java包特征
    java.lang: 基本类型数据包
    java.util: 数据结构包
    java.util.concurrent 并发包
    java.util.automic 原子包

  • 9.Object对象
    wait() //等待
    notify() //通知等待的线程

  • 10 java锁
    Object wait()让出锁阻塞、notify()、notifyAll()、join()等待当前线程终止
    synchronized 同步，可锁方法和代码块。专门用于互斥访问。非公平锁
    volatile 内存可见，可用于变量
    atomic 原子操作
    锁：
    Lock :接口用法如下

    ReentrantLock: 可重入锁（state用于标志重入 线程重复进入时，未加锁时，直接获取锁）。可选公平锁和不公平锁,不公平锁效率高。默认不公平。递归锁

    ReentrantReadWriteLock: 读写锁,默认不公平

    ReentrantReadWriteLock.ReadLock:线程进入读锁的前提条件: 没有其他线程的写锁。没有写请求或者有写请求，但调用线程和持有锁的线程是同一个

    ReentrantReadWriteLock.WriteLock:线程进入写锁的前提条件： 没有其他线程的读锁。没有其他线程的写锁

    自旋锁机制：实现会有一个参数限定最多持续尝试次数. 超出后, 自旋锁放弃当前time slice. 等下一次机会。

  • 13.jdk泛型
    Java中的泛型基本上都是在编译器这个层次来实现的。在生成的Java字节码中是不包含泛型中的类型信息的。使用泛型的时候加上的类型参数，会在编译器在编译的时候去掉。这个过程就称为类型擦除。擦除后，全部变为Object类型
    ? 不确定类型 (？ super String)确定是String的父类，可以插入内容
    T 确定类型的泛型
